ElasticStack-9-Elasticsearch篇之聚合分析入门

介绍Elasticsearch 聚合分析的功能，让大家了解其分类、组成，带领大家逐个了解、掌握每一个聚合 API 的使用方法和技巧，为后续 Kibana 使用打好基础。

1.聚合分析简介

1.1 什么是聚合分析?

  1. 1.搜索引擎用来回答如下问题:
    -请告诉我地址为上海的所有订单?
    -请告诉我最近1天内创建但没有付款的所有订单?

  2. 2.聚合分析可以回答如下问题:
    -请告诉我最近1周内每天的订单成交量是多少?(关注的不是订单，而是订单数量)
    -请告诉我最近1个月每天平均订单金额是多少?(关注的不是订单列表，而是订单金额)
    -请告诉我最近半年卖的最火的前5个商品是哪些?(关注的不是订单列表，而是订单里面的商品)

  3. 3.对于以上问题,假如我们没有用es时候,也是可以做的,但是对于每一个分析的话都是我们在客户端计算是非常的麻烦，es发现了这些问题,所以加上了聚合分析的能力。

  4. 4.聚合分析，英文为:Aggregation,是es除搜索功能外提供 的针对es数据做统计分析的功能。

ElasticStack-7-Elasticsearch篇之分布式特性介绍

1.分布式介绍及cerebro

1.1分布式介绍

  1. 1.es支持集群模式,是一个分布式系统,其好处主要有两个:
    -增大系统容量,如内存、磁盘、使得es集群可以支持PB级的数据。
    -提高系统可用性,即使部分节点停止服务,整个集群依然可以正常服务。

  2. 2.es集群由多个es实例组成
    -不同集群通过集群名字来区分,可通过cluser.name进行修改,默认为:elasticsearch
    -每个es实例本质上是一个JVM过程、且有自己的名字，通过node.name进行修改

7.文档分布式存储

  1. 1.文档最终会存储到分片上,如下图所示:
    -Document1最终存储在分片P1上,则在其副本R1上也存了一份

  2. 2.Document1是如何存储到分片P1的?选择P1的依据是什么?
    -Document1并不是随便选择到P1的,而是有一个文档到分片的映射算法。
    -映射算法目的:使得文档均匀分布在所有分片上,以充分利用资源。

  3. 3.算法
    -随机选择或者round-robin算法? 不可取:这种方法存储时候,文档存进去之后,我们需要读取的，比如：document1存储到p1了，我要通过这个文档id=1，如果是随机存储或者round-robin存储的获取，你是没法知道此文档1存储到那个分片上，两种选择:a.请求过来之后,在所有分片上都请求一下,有就返回,加入分片数有100个,浪费性能。b.开辟一个存储空间,存储文档到分片的映射关系。成本巨大(es是支持pb级别的数据存储,加入有上亿个文档,存储对应关系成本太高)。
    -我们采用的是：根据文档值试试计算对应的分片。

7.1 文档到分片的映射算法

  1. 1.es通过如下的公式计算文档对应的分片
    -shard = hash(routing)%number_of_primary_shards
    -hash算法保证可以将数据均匀地分散在分片中
    -routing是一个关键参数,默认是文档id,也可以自行指定。
    -number_of_primary_shards是主分片数。

  2. 2.该算法与主分片相关,这也是分片数一旦确定后便不能更改的原因。

7.2 文档创建的流程

1.Client向node3发起创建文档的要求
2.node3通过routing计算该文档应该存储在Shard1上,查询cluster state后确认主分片P1在node2上,然后转发创建文档的请求到node2
3.P1接收并执行创建文档请求后,将同样的请求发送到副本分片R1上
4.R1接收并执行创建文档请求后，通知P1成功的结果。
5.P1接收副本分片结果后,通知node3创建成功。
6.node3返回结果到Client端

  1. 1.脑裂问题,英文为split-brian,是分布式系统中的经典问题,如下图所示:
    -3个节点组成的集群,突然node1的网络和其他两个节点中断.node2与node3会重新选举master.比如node2成为了新master,此时会更新cluster state[cluster state 3’]
    -node1发现与其他节点连接不上时候,node1自己组成集群后,也会更新cluster state

  2. 2.同一个集群中有两个master,而且维护不同的cluster state,网络恢复后无法选择正确的master

  3. 3.解决方案为仅在可选举master-eligible节点数大于等于quorum时才可以进行master选举
    -quorum = master-eligible节点数/2+1，例如3个master-eligible节点时,quorum为2。
    -设定discovery.zen.minimum_master_nodes为quorum即可避免脑裂。

5.相关性算法

  1. 1.相关性算分是指文档与查询语句间的相关度,英文为:relevance
    -通过倒排索引可以获取与查询语句相匹配的文档列表,那么如何将符合用户查询需求的文档放到前列呢?
    -本质是一个排序问题,排序的依据是相关性算分。

  2. 2.相关性算分的几个重要概念如下:
    -Term Frequency(TF)词频,即单词在该文档中出现的次数,词频越高,相关性越高。
    -Document Frequency(DF)文档频率,即单词出现的文档数。
    -Inverse Document Frequency(IDF)逆向文档频率,与文档频率相反,简单理解为:1/DF,即单词出现的文档数越少,相似度越高。这个词在所在文档越少,说明这个单词越重要。
    -Field-length Norm 文档越短,相似度越高。

  3. 3.ES目前主要有两个相关性算分模型,如下：
    -TF/IDF模型
    -BM25模型 5.x之后的默认模型

4.分词介绍

  1. 1.分词是指将文本转换成一系列单词(term or token)的过程，也可以叫文本分析，在es里面称为Analysis,如下图所示:
  2. 2.es中负责分词的组件叫做分词器。分词器是es中专门处理分词的组件,英文名字为Analyzer,由3个部分组成。它的组成如下:
  • Character Filter:针对原始文本进行处理,比如去除html特殊标记符。
  • Tokenizer:将原始文本按照一定规则切分为单词。
  • Token Filters:针对tokenizer处理的单词进行再加工，比如转小写、删除或新增等处理。

7.中文分词

  1. 1.难点
    -中文分词指的是将一个汉字序列切分成一个一个单独的词。在英文中，单词之间是以空格作为自然分界符，汉语中词没有一个形式上的分界符。
    -上下文不同，分词结果迥异，比如交叉歧义问题，比如下面两种分词都合理:
    a.乒乓球拍/卖/完了
    b.乒乓球/拍卖/完了

12.分词使用说明

  1. 1.分词会在如下两个手机使用:
    -创建或更新文档时(Index Time),会对相应的文档进行分词处理。
    -查询时(Search Time),会对查询语句进行分词。

12.1索引时分词

  1. 1.索引时分词是通过设置Index Mapping中每个字段的analyzer属性实现的，如下:
    a.不指定分词时，使用默认standard

12.2查询时分词

  1. 1.查询时分词的指定方式有如下几种:
    -查询的时候通过analyzer指定分词器
    -通过index mapping设置search_analyzer实现
  2. 2.一般不需要特别指定查询时分词器，直接使用索引时分词器即可，否则会出现无法匹配情况。

    12.3 分词的使用建议

  3. 1.明确字段是否需要分词，不需要分词的字段就将type设置为keyword,可以节空间和提高性能。

  5. 动手测试。
