Web服务器启动读取数据库参数

前言

今天，项目需要在服务器一启动就加载数据库参数，之前考虑到使用监听器，但是在考虑到如何读取数据库的数据时候出现了问题，我在监听器里面配置service，然后service调用dao，所以就出现了空指针异常，后面想到了用Servlet里面的init方法

内容

自启动，在web.xml文件配置如下：
  1. 标记web容器是否在启动的时候就加载这个servlet
  2. 当值为0或者大于0时，表示web容器在应用启动时就加载这个servlet
  3. 当是一个负数时或者没有指定时，则指示容器在该servlet被选择时才加载,正数的值越小，启动该servlet的优先级越高
systemInit如下:

重启web容器，在控制台可以看到执行的信息。
然后我们来修改我们的代码，让这个Servlet可以读取数据库的内容，并进行相关的处理。
我操作数据库用的是spring的注入，访问数据库是以bean注入的方式，在 SystemInit里对应的service注入service的实现，读取数据库数据，其实现在的关键就是得到这个service的实例化对象，

//service
@Service("systemInitService")
public class SystemInitServiceImpl implements SystemInitService{

    @Autowired
    private ParamDao paramDao;
    @Override
    public List<ParamVO> getAllParam() {
        return paramDao.getAll();
    }
}  
//这个Servlet实现  
public class SystemInit extends HttpServlet {
    /**
     * 
     */
    private static final long serialVersionUID = 1L;

    public SystemInit() {
        super();
    }

    public void init() throws ServletException {
      ServletContext sc= getServletContext(); 
      WebApplicationContext ac = WebApplicationContextUtils.getRequiredWebApplicationContext(sc);
      SystemInitService service = (SystemInitService)ac.getBean("systemInitService");
      List<ParamVO> allParam = service.getAllParam();
      for (ParamVO paramVO : allParam) {
          SystemInitParamUtil.paramList.add(paramVO);
      }
    }
}

systemInitService也可以是在spring配置文件里面配置的当然以上是注解

解决Myeclipse10内存溢出

错误

启动项目，报错：
java.lang.OutOfMemoryError: PermGen space

解决

1. 在Myeclipse根目录下修改myeclipse.ini：

#utf8 (do not remove)
-startup
../Common/plugins/org.eclipse.equinox.launcher_1.2.0.v20110502.jar
–launcher.library
../Common/plugins/org.eclipse.equinox.launcher.i18n.win32.win32.x86_64_4.2.0.v201201111650
-install
C:\Users\xxx\AppData\Local\MyEclipse\MyEclipse 10
-vm
C:\Users\xxx\AppData\Local\MyEclipse\Common\binary\com.sun.java.jdk.win32.x86_64_1.6.0.013\bin\javaw.exe
-configuration
C:\Users\xxx\AppData\Local\MyEclipse\MyEclipse 10\configuration
-vmargs
-Xmx1024m
-XX:MaxPermSize=512m
-XX:ReservedCodeCacheSize=128m
-Dosgi.nls.warnings=ignore

2. myeclipse.ini里配置后
  1. 设置Default VM Arguments
    在myEclipse中，打开Windows-> Preferences->Java->Installed JREs->点击正在使用的JDK->Edit->Default VM Arguments文本框中输入：-Xms64m -Xmx256m
  2. 如果是web工程还报内存溢出，可能就要设置tomcat内存。
    （1）tomcat的安装根目录%tomcat%bin/catalina.bat文件，SET JAVA_OPTS 一行；
    （2）myeclipse配置web服务器配置 Window—Preferences—Myeclipse—Servers—tomcat 5.x—JDK的Optional Java VM arguments配置为：
    -Xms256m -Xmx512m -XX:MaxNewSize=256m -XX:MaxPermSize=256m
