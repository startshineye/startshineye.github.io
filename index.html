动态添加信息就是相关js的书写:如下:

<script type="text/javascript">
      mui.init();


      mui.plusReady(function(){
          //获取当前webView,刷新当前信息
          var meWebview = plus.webview.currentWebview();
          meWebview.addEventListener("show",function(){
              refreshUserInfo();
          })

          //刷新,加载用户信息
          function refreshUserInfo(){
              var user = app.getGlobalUserInfo();
              console.log(JSON.stringify(user));
              if(null != user){
                 var faceImage = user.faceImage;
                 var nickname = user.nickname;
                 var username = user.username;

                 var span_nickname = document.getElementById("span_nickname");
                 var span_username = document.getElementById("span_username");

                 span_nickname.innerHTML = nickname;
                 span_username.innerHTML = username;

                 if(app.isNotNull()){
                     var img_my_face = document.getElementById("img_my_face");
                  img_my_face.src = app.imgServerUrl + faceImage;
                 }
              }
          }
      })
  </script>

前言

爬虫宕机需要进行监控,防止项目挂掉之后未有及时的应急处理.

内容


从以上可以知道,监控分为硬件监控和软件系统监控,硬件监控用Ganglila,软件监控用Zookeper

1.Zookeper

zk采用多节点部署,每一个节点叫做znode,每一个节点的路径结构是唯一确定的

Znode，利用zk的短暂Znode和持久化的Znode也有属性来实现监控功能。

2 监控结构

  1. client1 client2 … 代表我们的爬虫服务
  2. 爬虫生命周期监控:每启动一个爬虫服务client时候 我们向zookeeper节点(GroupMembers)注册一个临时节点,一旦节点挂掉和停止了,临时节点(client1)就会消失,此时我们只需要一个watcher来监控所有节点,只要有节点消失掉我们watcher就会知道
  3. 正常情况下,临时节点不会消失,只有当在有一个节点挂掉了,临时节点消失,我们可以触发监控程序,知晓哪个节点挂掉了

  4. 根据以上结构,我们需要做一个watcher服务,用于监控爬虫服务,再建立一个client到zookeeper注册服务(Register)

  1. 设置每个页面抓取时间间隔,降低被封概率,之前我们已经采取了措施,就是设置抓取时间间隔设置成3秒,5秒,一般来说 抓取时间超过1秒才算合理

  2. 设置成3 5秒抓取时间比较固定,一般来说,网站能够分析出你抓取的频率,鉴定出是爬虫,我们设置随机数（1 2 3 4 5）秒 总是变化中的 让网站相信 我们是合法用户

  3. 我们在开始爬虫方法中读取配置文件的时间改成随机数:Math.random

  1. 缺点:此种方式在【网站监测到同一个ip访问过多情况下 也会封ip情况】 并不能解决,就是不让一天访问过多,
