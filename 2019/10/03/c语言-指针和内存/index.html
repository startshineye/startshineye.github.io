3.指针与内存

计算机中的数据表示方式、内存的管理、变量与指针的本质、函数指针以及指针指向的数据访问

3.1计算机中的数据表示方法

  1. 1.计算机中最小的单位叫做:字节(Byte)
  2. 2.1个字节是8个二进制位:1Byte = 8bit

  3. 3.为什么我们需要用“二进制”表示数据。由于我们现在使用的是电子计算机。电子计算机是逻辑电路元件做成的。因为计算机中是用电的,我们的电流状态只有2钟状态。一种是高电位,高电位对应的电路会通,我们接通一个灯泡后会亮,那么灯亮了就是1；低电位,电路不通,此时接入一个灯泡后灯不会亮。那么认为低电位时候,电流就是0。

  4. 4.高电位是1,低电位是0,那么计算机处理再复杂的数据也脱离不了这些1和0；所以不管我们听歌、看电影、玩游戏等。在计算机中由cpu处理的数据全都是101010等组成。人对于这些数据是不敏感的,人和计算机进行交互,其实就是十进制和二进制进行变换。

  5. 5.在计算机中如果全部用二进制表示将会有很多很多1010…，挺麻烦，所以引入了十六进制。 十进制、二进制、十六进制之间表示关系如下:
  6. 6.计算机计算时候是按照二进制,展示给人看的时候是十进制。我们编程时候使用十六进制。

3.2内存管理

知道计算机的数据表示方法后,那我们需要知道我们的内存到底是什么呢?

  1. 1.内存由操作系统统一管理。
  2. 2.一个字节是8个二进制位,我们在计算几种插入2个内存条,4个内存条。插一根4G内存条和2根2G内存条其实是等效的。计算机中不管插入了多少内存条，他会把内存看成一个整体来计算内存大小。
  3. 3.但是计算机并不是插多少根内存条就可以的,比如:32位的计算机不管插入多少内存条,最大使用内存是4G。
  4. 4.为什么32位操作系统只能使用4G内存呢?因为32位的电脑上,cpu的地址总线是32位。也就是操作系统的寻址空间是32位。32位指的是:给内存编号只能编到32个二进制位。这个编号就是类似于我们接到的门牌号码一样。地址总线有多少根那么存在状态就有对应的组合。每一根总线有2种状态。32根总线就有2的32此房个状态。其中一个编号就能代表计算机中内存的最小存储单位字节。则可以存储2的32次方的字节。
  5. 5.2的32次方个字节等于多少呢?
  6. 6.64为计算机存储2的64位,此时64位置系统下的内存足够使用了。
  7. 7.编号=唯一的内存字节的地址。
  8. 8.应用程序由操作系统来调用的。

  9. 9.main()函数就是所有函数的入口。

  10. 10.计算机内存分布情况如下:

    a.低位内存方便管理又进行了分片:最低位的 代码段(c语言代码-函数编译之后存到磁盘,一旦运行此程序后,代码编译后的二进制数据加载到内存中,此部分内存就是代码段)。
    b.声明的一些全局变量或者声明的一些常量:const int i;int global;这些存放到数据段。
    c.一般数据段内存地址编号通常会比代码段内存大小要大,代码段和数据段都是我们操作系统预先规划好的，不管有多少应用程序,代码段和程序段都是在低位的。
    d.中间绿色区域为:自由可以分配的内存。此区域越大调用的资源就越多。
    e.除了以上内存外:计算机还有堆内存、栈内存。

3.3.1 变量的本质

变量的本质就是内存,变量名它只是一个代号。

  1. 1.a变量当main函数被调用的时候、通过栈内存空间来保存我们这个a,a只是一个代号,代表了0x7fffffffe468内存空间而已。
  2. 2.我们写程序时候,计算机处理的都是二进制的数据。而我们的二进制数据肯定是存在内存里面cpu才能取出来。cpu从内存中取数据,那么cpu必须知道从哪个内存中取数据。而变量就起到了一个表示作用,用来告诉我们cpu要从什么地方去取数据。或者把数据写到什么地方去。
    3.3.2 指针的本质
  3. 1.c语言中所有的变量都有类型;int 类型就保存整数,double类型就保存的双精度的浮点数,指针他也是一种类型,他保存的就是内存的地址。
  4. 2.指针本质上就是地址。
