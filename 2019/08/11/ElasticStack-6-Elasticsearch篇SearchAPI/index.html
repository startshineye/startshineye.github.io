ElasticStack-6-Elasticsearch篇SearchAPI

本章会讲解搜索特性，详细讲解 Search API 的组成和分类，带领大家逐个了解、掌握 API 的使用方法和技巧。

1.Search API概述

  1. 1.Search API实现对es中存储的数据进行查询分析,endpoint为_search,如下所示:

  2. 2.查询主要有两种形式
    a.URI Search
    -操作简便,方便通过命令行测试
    -仅包含部分查询语法
    b.Request Body.Search
    -es提供的完备查询语法Query DSL(Domain Specific Language)

2.URISearch详解与演示

  1. 1.通过url query参数来实现搜索,常见参数如下:
    -q 指定查询的语句，语法为 Query String Syntax
    -df q中不指定字段时默认查询的字段，如果不指定，es会查询所有字段
    -sort 排序
    -timeout 指定超时时间,默认不超时
    -from,size用于分页

2.1.1 基本查询

  1. 1.term与phrase
    -alfred way 等效于alfred OR way
    -“alfred way”词语查询，要求先后顺序

  2. 2.泛查询
    -alfred等效于在所有字段去匹配该term

  3. 3.指定字段
    -name:alfred

  4. 4.Group分组设定,使用括号指定匹配的优先级规则
    -(quick Or brown) AND fox
    -status:(active OR pending) title:(full text search)

2.2.2 布尔操作符
  1. 1.布尔操作符
    -AND(&&),OR(||),NOT(!)
    a.name:(tom NOT lee)–表示name包含tom 不包含 lee
    b.注意大写,不能小写

-+-分别对应must和must_not
a.name:(tom+lee-alfred)–表示name一定包含lee一定不包含alfred可以包含tom
b.name:(lee && !alfred) || (tom && lee && !alfred)
c.+在url中会被解析为空格,要使用encode后的结果才可以,为%2B

  1. 2.实例
    a.获取username字段中包含alfred但是不包含way的
    -GET test_search_index/_search?q=username:(alfred NOT way)
