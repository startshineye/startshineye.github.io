ElasticStack-5-Elasticsearch篇之Mapping设置

本章会讲解 Elasticsearch 中数据建模的基础–Mapping，即如何定义数据字段和类型。让大家熟悉 mapping 中常见的配置项，也会讲解 dynamic mapping 和 template 的相关知识。

1.mapping简介

  1. 1.类似于数据库中表结构定义，主要作用如下:
    -定义Index下的字段名(Field Name)。
    -定义字段的类型，比如数值型、字符型、布尔型等。
    -定义倒排索引相关的配置，比如是否索引、记录position等。

  1. 2.Mapping中的字段类型一旦设定后，禁止直接修改，原因如下:
    a.Lucene实现的倒排索引生成后不允许修改(为了提高效率)
    b.原因如果mysql中心定义表结构后，重新定义表结构会更新整个表
    c.如果硬要改的话需要:重新建立新的索引，然后做reindex操作

    -strict文档不能写入,报错

5.index参数说明

  1. 1.index
    -控制当前字段是否索引,默认为true,即记录索引,false不记录索引,即不可搜索。 用于希望有些字段不让人查询、比如一些敏感信息：身份证号等

  2. 2.设置index为false的好处
    -a.实现不被检索需求
    -b.不存倒排索引了、所以节省空间

6.index_options参数说明

index_options用于指定倒排索引创建时候,记录索引的哪些内容。

  1. 1.index_options用于控制倒排索引记录的内容，有如下4种配置
    -docs 只记录doc id(文档id)
    -freqs 记录doc id(文档id)和term frequencies(单词出现频率)
    -freqs 记录doc id和term frequencies和term position(单词位置)
    -offsets记录doc id,term frequencies,term position,character ofsets

  2. 2.text类型默认配置为postions,其他默认为:docs

  3. 3.记录内容越多,占用空间越大

8.mapping数据类型

我们进入es浏览章节:Elasticsearch Reference [7.1] » Mapping » Field datatypes就可以查看mapping的数据类型。

8.1 核心数据类型(Core datatypes)

官网如下:

  1. 1.字符型
    -text(支持分词)和keyword(不支持分词)
  2. 2.数值型
    -long, integer, short, byte, double, float, half_float, scaled_float
  3. 3.日期类型
    -date
  4. 4.布尔类型
    -boolean
  5. 5.二进制
    -binary
  6. 6.范围类型
    -integer_range, float_range, long_range, double_range, date_range

11.Dynamic Mapping简介

  1. 1.允许根据es自动识别的数据类型、字段等来动态设定字段类型，可以实现如下效果:
    -将所有字符串类型都设定为keyword类型，即默认不分词(不设置的话默认es会设置为分词的text,里面有倒排索引、暂用空间)。
    -所有以message开头的字段都设定为text类型,即分词
    -所有以long_开头的字段都设定为long类型
    -所有自定匹配为double类型的都设定为float类型、以节省空间。

  2. 2.API如下所示:

  3. 3.匹配规则一般有如下几个参数
    -match_mapping_type匹配es自动识别的字段类型，如:boolean,long,string等
    -match,unmatch匹配字段名
    -path_match,path_unmatch匹配路径

  4. 4.字符串默认使用keyword类型
    -es默认会为字符串设置为text类型,并增加一个keyword的子字段

  5. 5.以message开头的字段都设置为text类型

  6. 6.double类型设置为float,节省空间

12.Dynamic Mapping简介

有时候我们创建mapping时候,字段比较多的时候,我们一个个创建比较麻烦,我们可以通过先创建文档、然后获取mapping,再拷贝出来修改自定义mapping

  1. 1.自定义mapping操作步骤如下:
    1.写入一条文档到es的临时索引中,获取es自动生成的mapping
    2.修改步骤哦1得到的mapping,自定义相关配置
    3.使用步骤2的mapping创建实际所需索引
