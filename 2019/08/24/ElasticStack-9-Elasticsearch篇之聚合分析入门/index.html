ElasticStack-9-Elasticsearch篇之聚合分析入门

介绍Elasticsearch 聚合分析的功能，让大家了解其分类、组成，带领大家逐个了解、掌握每一个聚合 API 的使用方法和技巧，为后续 Kibana 使用打好基础。

1.聚合分析简介

1.1 什么是聚合分析?

  1. 1.搜索引擎用来回答如下问题:
    -请告诉我地址为上海的所有订单?
    -请告诉我最近1天内创建但没有付款的所有订单?

  2. 2.聚合分析可以回答如下问题:
    -请告诉我最近1周内每天的订单成交量是多少?(关注的不是订单，而是订单数量)
    -请告诉我最近1个月每天平均订单金额是多少?(关注的不是订单列表，而是订单金额)
    -请告诉我最近半年卖的最火的前5个商品是哪些?(关注的不是订单列表，而是订单里面的商品)

  3. 3.对于以上问题,假如我们没有用es时候,也是可以做的,但是对于每一个分析的话都是我们在客户端计算是非常的麻烦，es发现了这些问题,所以加上了聚合分析的能力。

  4. 4.聚合分析，英文为:Aggregation,是es除搜索功能外提供 的针对es数据做统计分析的功能。
    -功能丰富,提供Bucket、Metric、Pipeline等多种分析方式，可以满足大部分的分析需求。
    -实时性高,所有的计算结果都是及时返回,而hadoop等大数据系统一般都是T+1级别的

1.2 聚合分析的API

  1. 1.聚合分析作为search的一部分,api如下所示:

    -HttpMethod使用GET、POST都可以
    -聚合分析也是在search下面的
    -关键词使用与query同级别的aggs,aggs是由于Aggregations太长后的简洁写
    -自定义聚合名称aggregation_name、返回结果时候是把所有结果放到这个名字里面、名字是一个jsonObject里面又需要指定:aggregation_type-用来定义使用哪种aggregation、aggregation_body用来指定aggregation_type相关的配置。
    -一个aggs里面是可以同时有多个聚合分析的、可以有多个,比如上图的aggregation_name_2

  2. 2.我们主要学习aggregation_type(有哪些聚合类型)以及aggregation_body(对于每一个aggregation_type的限制)

  3. 3.例子:请告诉我公司目前在职人员工作岗位的分布情况?

    -aggs-关键词
    -aggregation_name-people_per_job
    -aggregation_type-terms
    -aggregation_body-field为关键词
    -返回时候失败结果返回到自定义聚合名称aggregation_name里面，返回的buckets就是返回结果、doc_count就是返回文档个数

1.3 聚合分析的分类

  1. 1.为了便于理解,es将聚合分析主要分为如下4类
    -Buckets,分桶类型,类似SQL中的Group BY 语法。
    -Metric,指标分析类型,如计算最大值,最小值,平均值等等。
    -Pipeline,管道分析类型,基于上一级的聚合分析结果进行再分析
    -Matrix,矩阵分析类型-是es后面加入的

2.Metric聚合分析

2.1 Metric聚合分析类别

  1. 1.主要分为如下两类：
    -单值分析,只输出一个分析结果
    -min,max,avg,sum
    -cardinality(计算数目的,理解成distinct count)
    -多值分析,输出多个分析结果
    -stats,extended stats(多样分析,包含最小值,最大值)
    -percentile,percentile rank(百分位数统计)
    -top hits(取排在前面的hit列表)
