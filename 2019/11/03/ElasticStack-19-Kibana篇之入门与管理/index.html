1.Management

1.1 创建Index Paterns

Kibana uses index patterns to retrieve data from Elasticsearch indices for things like visualizations.(Kibana使用索引模式从Elasticsearch索引中检索可视化数据。)，如果不设置Index Paterns的话,ES是不知道怎样从es中取数据。

  1. 1.Step 1 of 2: Define index pattern(其实就是一个通配符)
  2. 2.Step 2 of 2: Configure settings

    Custom index pattern ID:kibana在存储时候,会生成对应id
1.2 查看Index Paterns详情

点击左边的Index可以从右边看到对应的Index详情。
-Time Filter field name: @timestamp:指明过滤器是时间戳。
-5角星(Set as default index):把对应的索引设置为默认的,设置后其对应的左边的index pattern列表会变化。
-刷新按钮(Refresh field list):如果索引有了新的字段,kibana不会自动探索,而是需要通过refresh把新的字段加载起来。

再讨论Index的属性:Fields、Scripted fields、Source filters
-Fields:索引的所有字段,点击进去后以列表形式展示。
