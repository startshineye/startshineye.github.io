websocket(6)-websocket实现单聊

用websocket实现单聊

1.java后台

1.websocketconfig

@Configuration
@EnableWebSocketMessageBroker
public class WebsocketConfig implements WebSocketMessageBrokerConfigurer {
@Override
public void registerStompEndpoints(StompEndpointRegistry registry) {
    registry.addEndpoint("websocket").withSockJS();
}

@Override
public void configureMessageBroker(MessageBrokerRegistry config) {
   config.enableSimpleBroker("/topic","/chat");
   config.setApplicationDestinationPrefixes("/app");
}
}

2.controller

@Controller
public class V3SingleChatController {

@Autowired
private WebsocketService websocketService;

@MessageMapping("/v3/singlechat")
public void singleChat(InMessage message){
    websocketService.sendSingleChatMessage(message);
}
}

3.service

@Service
public class WebsocketService{
@Autowired
private SimpMessagingTemplate template;

public void sendTopicMessage(String dest,HelloMessage message){
    template.convertAndSend(dest,new Greeting(message.getName()));
}

public void sendSingleChatMessage(InMessage message) {
    System.out.println("dest:"+"/chat/single/"+message.getTo());
    template.convertAndSend("/chat/single/"+message.getTo(),
            new OutMessage(message.getFrom()+"发送:"+message.getContent())
            );
}
}
2.前端页面

1.app.js 

var stompClient = null;

function setConnected(connected) {
  $("#connect").prop("disabled", connected);
  $("#disconnect").prop("disabled", !connected);
  if (connected) {
    $("#conversation").show();
  } else {
    $("#conversation").hide();
  }
  $("#greetings").html("");
}

function connect() {
  var from =  $("#from").val();
  var socket = new SockJS('/websocket');
  stompClient = Stomp.over(socket);
  stompClient.connect({}, function (frame) {
    setConnected(true);
    console.log('Connected: ' + frame);
    stompClient.subscribe('/chat/single/'+from, function (greeting) {
        showGreeting(JSON.parse(greeting.body));
    });
  });
 }

function disconnect() {
if (stompClient !== null) {
    stompClient.disconnect();
}
  setConnected(false);
  console.log("Disconnected");
}

function sendName() {
stompClient.send("/app/v3/singlechat", {}, JSON.stringify({'content': $("#content").val(), 'to':$("#to").val(), 'from':$("#from").val()}));
}

function showGreeting(message) {
$("#greetings").append("<tr><td>" + message.content + "</td></tr>");
}

$(function () {
$("form").on('submit', function (e) {
    e.preventDefault();
});
$( "#connect" ).click(function() { connect(); });
$( "#disconnect" ).click(function() { disconnect(); });
$( "#send" ).click(function() { sendName(); });
});

1.是什么?

  2. 提供了一个连贯的跨浏览器的JavaScriptAPI，在浏览器和Web服务器之间创建了一个低延迟，全双工，跨域的通信通道
  3. 在底层SockJS首先尝试使用本地WebSocket。如果失败了，它可以使用各种浏览器特定的传输协议，并通过类似WebSocket的抽象方式呈现它们
  4. SockJS旨在适用于所有现代浏览器和不支持WebSocket协议的环境。

linux(17)-管道符

1.前言
  1. Linux所提供的管道符“|”将两个命令隔开，管道符左边命令的输出就会作为管道符右边命令的输入。连续使用管道意味着第一个命令的输出会作为第二个命令的输入，第二个命令的输出又会作为第三个命令的输入，依此类推。
  2. 命令1 | 命令2 –>命令1的正确输出作为命令2的操作对象
2.命令
  1. netstat命令可以查看所有网络连接
  2. grep查询命令
  3. ps查看进程命令 -aux选项可以查看整个系统的进程

利用管道符查看Tomcat是否启动：
ps -ef|grep ‘tomcat’

ps 查看当前终端的进程
ps -aux 查看整个系统的进程
ps -aux | grep java 通过管道符查看java进程

linux(16)-多命令执行符

多命令执行符

分号(:)–>多个命令顺序执行

逻辑与(&&) –>当命令1执行正确,命令2才会执行,当命令1执行错误,命令2不会执行

逻辑或 -> ||
当命令1执行不正确,才会执行命令2,当命令1执行完毕,命令2不会执行

linux(15)-yum源码包安装

1.源码包和rpm包区别

概念上:rpm包是编译好的包,而源码包是没有编译好的包,是可以看到源码的
安装位置上:
Rpm包的默认安装位置如下: 这只是仅仅默认安装位置,但是80%以上的安装包不是这样的安装目录。
源码包一定要指定安装路径,一般有以下规范,但是你自己也可以不按照以下:
/usr/local/软件

2. 启动
  1. Rpm包启动方法:service 包名 start
    关闭方法:service 包名 stop
  2. apachectl start
3.源码包安装步骤:

1． 解压
$ tar –zxvf httpd-2.2.34
2． 制定安装位置并安装: /usr/local/软件
比如:安装apache
$./configure –prefix=/usr/local/apache
3.编译
$ make install

如果重新安装:切记一定要记住清除编译 make clear 清除编译缓存

  1. 启动
    进入apache安装源码目录下,进入安装说明:
    $ vi INSTALL

查看到安装说明,一般软件代码都有此说明的

linux(14)-yum命令

1.恢复在线yum

  1. yum list

  2. yum search 关键字(包名)

    比如:搜索apache
    yum search httpd

  3. -y(自动回答yes)
    比如安装zsh 半途会提示你输入yes或No 添加y会自动添加yes

  4. yum update 包名 更新某一软件

    yum remove 包名->卸载软件包
