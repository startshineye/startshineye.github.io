java放射(六)-反射调用main方法

用反射方式执行某个类中的main方法

1.目标

写一个程序，这个程序能够根据用户提供的类名，去执行该类中的main方法。用普通方式调用完后，为什么要用反射方式去调用呢？

2.问题

启动java程序的main方法的参数是一个字符串数组,即是public static void main(String[] args),通过反射方式来调用这个mian方法时，如何为invoke方法传递参数呢？jdk1.5的语法，整个数组是一个参数，jdk1.4语法，数组中的每一个元素对应一个参数,当把一个字符串数组作为参数传递给invoke方法时候，javac底层会按照哪种语法进行处理呢？jdk1.5肯定要兼容jdk1.4的语法，会按照jdk1.4的语法进行处理，即把数组打散成为若干个单独的参数。所以，在给main方法传递参数时候，不能使用代码mainMethod.invoke(null,new String[]{“xxx”}),javac只把他当做jdk1.4的语法进行理解,而不把它当做jdk1.5的语法理解，因此会出现类型不匹配的问题。

3.解决办法
  1. mainMethod.invoke(null,new Object[]{new String[]{“xxx”});
  2. mainMethod.invoke(null,(Object)new String[]{“xxx”});编译器会做特殊处理，编译时不把参数看做数组对待，也就不会数组打散成若干个参数。

Constructor类

  2. 得到某个类中所有构造方法：
    Constructor[] constructorArr = Class.forName(“类名”).getConstructors();
  3. 创建实例对象
    通常方式：String str = new String(new StringBuffer(“abc”));

实现资源转发-javaServer16

我们再来看这样一种情况，例如有这样一个功能，当用户登录成功之后跳转到显示员工列表页面，登录功能需要服务器端调用WebApp的LoginServlet进行处理，当LoginServlet执行结束登录成功后需要在同一个请求当中显示员工列表信息，而显示员工列表信息则需要执行另一个Servlet，例如EmpListServlet，这就相当于在同一个请求当中，先后执行了两个Servlet，先执行LoginServlet，然后再跳转到EmpListServlet继续执行，这个我们应该怎么做呢？
有人说，在LoginServlet中service方法执行结束的时候，由WebApp开发人员手动调用EmpListServlet中的service方法不就行了吗。这当然是不行的，因为Servlet对象的创建，包括Servlet对象上方法的调用，都是由Web服务器来管理的，WebApp开发人员无权干涉。那该怎么办呢？只好把这个Servlet对象上service方法的真正调用交给Web服务器来完成了。
那我们把这种机制定义为转发机制，转发过程中必须保证是在同一个请求当中，为了保证LoginServlet和EmpListServlet在同一个请求当中，必须保证LoginServlet和EmpListServlet共享同一个request对象和同一个response对象。

自己写一个apache服务器(十七)-Servlet对象实现单例-javaServer15

Servlet对象实现单例-javaServer15


从上图可以看出，每一次浏览器客户端发送请求的时候，WEB服务器都会通过反射机制创建一个新的Servlet对象，当多线程同时并发访问的时候，在WEB服务器的JVM中会有大量的Servlet对象，使JVM的堆内存压力增大，耗费占用大量的堆内存空间。并且每一次都创建Servlet对象，显然效率是比较低的，怎么做既可以节省堆内存的开销又可以提高程序的执行效率呢？当然，此时你肯定想到了单实例的方式，也就是说当浏览器客户端第一次发送请求的时候创建Servlet对象，将Servlet对象和对应的请求URI存放到缓存当中，当浏览器客户端第2+次发送请求的时候直接从缓存当中获取请求URI对应的Servlet实例，直接调用Servlet对象的service方法。这样做我们就可以达到既节省了内存的开销，又提高了程序的执行速度。接下来我们做一个简单的设计，如下图所示：
