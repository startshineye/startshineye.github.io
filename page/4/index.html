正文

一:String,StringBuffer,StringBuilder之间的区别与联系

1. 区别与联系
  • [相同点]String,StringBuffer,StringBuilder 底层都是采用char数组
  • [不同点]String是定长的字符串: 是存储在方法区中的常量池之中,任何对字符串的修改操作都会创建新的对象,不适合做字符串的拼接
  • [不同点]StringBuffer,StringBuilder是可变长的
    1. StringBuffer,StringBuilder 默认的数组初始化长度是16
    2. StringBuffer,StringBuilder 如果长度不够的话,需要对数组进行扩容,底层使用的数组拷贝的方式对数组进行扩容 System.arraycopy()
    3. 该方法前面用native修饰,属于本地方法调用 (JNI Java Native Invoke),表示调用了其它语言编写的dll文件
    4. 关于优化:在做大数据量字符串拼接的时候,可以指定数组的初始化长度,减少扩容的次数,减少数组拷贝的次数
    5. StringBuilder是JDK1.5之后出现的（比较新）
      StringBuffer是JDK1.0就出现了（比较老，并发性不行，每一个方法上都有synchronized，效率较低，因为有些情况下synchronized没必要编写，例如：局部变量就不存在线程安全问题。）
2.源码

二:线程安全问题

1.发生条件
  1. 多个线程
  2. 共享资源
  3. 对共享资源涉修改操作
    以上三个条件都满足的情况下,才需要考虑线程安全的问题
2.解决方案
  • 多线程共享：堆内存、方法区内存(实例变量、静态变量会存在线程安全问题，实例变量在堆中，静态变量在方法区中)
    栈内存独立，一个线程一个栈。(结论：局部变量不存在线程安全问题，因为局部变量不共享)
  • 解决方法
    方案1：尽量使用局部变量
    方案2：如果是实例变量，我们可以让对象变成多例的也可以解决线程安全问题。
    方案3：如果是实例变量，我们还可以添加synchronized进行控制保证线程安全
    多例：耗费堆内存
    添加synchronized影响并发

三:总结

1. 字符串拼接速度及原因
  • StringBuilder>StringBuffer>String
  • 原因
    基于以上分析可以总结出以下原因:
    1. String是不可变的（final），每次改变其变量值，生成一个新的对象，然后将变量引用指向新对象；速度慢。
    2. StringBuffer直接操作对象指向的引用，无需产生新对象，速度很快；它是线程安全的，在维护多线程的同步等也会消耗一点性能。
    3. StringBuilder是jdk5之后新增的，用法与StringBuffer完全一致，但它是线程不安全，在单线程中最佳，因为其不需要维护线程的安全，因此是最快的

CTI简单概述

CTI技术

什么是CTI技术

CTI是计算机电信集成（Computer Telecommunication Interation）的英文缩写，它覆盖了数据通信网络及传统语音通信网络的内容，是由传统的计算机电话集成技术(Computer Telephone Integration)发展演变而来，是指计算机与电信的集成技术，在国外将其称为CT(Computer telephony)技术，即通常的计算机电话技术。它随着电信技术和计算机技术的发展而产生发展起来。随着两者的逐步融合，在计算机领域中引入了通信技术，在电信设备中也增加了计算机技术的应用，这就诞生了CTI,这个横跨电信和计算机两大领域的新技术

CTI的优点

  1. CTI技术可以可以把电话的通信功能和计算机的处理处理，控制功能融合在一起
  2. 实现通信增值服务，从而满足不同用户的需求

CTI主要应用

主要应用在下面的几个方面：
呼叫中心(Call Center)、交互式语音应答(IVR)、统一消息、小型办公等。其应用领域设计到银行、保险、企业等等（比如电话销售、客服服务等等）。呼叫中心是 CTI技术应用的最重要的领域。

目前， CTI技术已经与Internet结合，多媒体呼叫中心的时代已经来临,统一通信的技术已经形成

Redis入门

一:Redis简介

Redis是一个开源，先进的key-value存储，并用于构建高性能，
可扩展的Web应用程序的完美解决方案。

1.主要特点
  • Redis数据库完全在内存中，使用磁盘仅用于持久性。
  • 相比许多键值数据存储，Redis拥有一套较为丰富的数据类型。
  • Redis可以将数据复制到任意数量的从服务器。
2.Redis优势
  1. 异常快速：
    Redis的速度非常快，每秒能执行约11万集合，每秒约81000+条记录。
  2. 支持丰富的数据类型：
    Redis支持最大多数开发人员已经知道像列表，集合，有序集合，散列数据类型。这使得它非常容易解决各种各样的问题，因为我们知道哪些问题是可以处理通过它的数据类型更好。
  3. 操作都是原子性：
    所有Redis操作是原子的，这保证了如果两个客户端同时访问的Redis服务器将获得更新后的值。
  4. 多功能实用工具：
    Redis是一个多实用的工具，可以在多个用例如缓存，消息，队列使用(Redis原生支持发布/订阅)，
    任何短暂的数据，应用程序，如Web应用程序会话，网页命中计数等。
