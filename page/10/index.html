oracle创建表空间,用户,授权并导入dmp文件

前言

在很多项目特别是大型项目开发中都需要熟练操作oracle创建表空间,用户,授权并导入dmp文件。并且很多数据库都需要创建表空间。

内容

数据库

在mysql中创建数据库只需要命令:
create database 数据库名
但是在ORACLEl中创建数据库:
打开tnsnames.ora文件
创建ORCL数据库:

表空间

  1. Oracle数据库被划分为称为”表空间”的逻辑区域–形成ORACLE的逻辑结构.一个Oracle数据库能够有一个或多个表空间,而一个表空间则对应一个或者多个物理的数据库文件
  2. 表空间是ORACLE数据库恢复的最小单元,容纳着许多数据库实体:如：表,视图,索引,聚簇,回退和临时段等
  3. 每一个Oracle数据库均有SYSTEM表空间(这是数据库创建时自动创建的),一个小型应用的Oracle数据库通常仅包括SYSTEM表空间,然而一个稍大应用Oracle数据库采用多个表空间

    2.在SQL Window下输入以下命令查看表空间及其硬盘上的位置
  5. 创建表空间
    在创建表空间之前需要查看dmp文件对应的表空间
    用notepade++打开dmp文件,ctrl+f打开搜索框输入tablespace找到表空间:CCDB_TBS_IDX,CCDB_TBS_DAT

    创建表空间
    create tablespace CCDB_TBS_IDX datafile ‘D:\ORACLE\APP\ORACLE\ORADATA\XE\CCDB_TBS_IDX.DBF’ size 200m autoextend on next 100m maxsize unlimited;

JAVA环境变量找不到

前言

把Oracle自带的weblogic给卸载了，然后在cmd下输入java -version，发现报错了：
Error: could not open `C:\Java\jre7\lib\amd64\jvm.cfg

内容

  1. JAVA_HOME的环境变量值根本不是这个目录，到CMD命令窗口下，用echo %JAVA_HOME%，也显示正常，但是java -version时，显示java的home目录仍然是C:\Java\jre7，百度后原来JDK如果执行过安装程序后，会在HKEY_LOCAL_MACHINE\SOFTWARE\JavaSoft\Java Runtime Environment\1.7.0_75\MSI\ 留下印记，里面有一项INSTALLDIR记录的还是当时安装的旧路径，如下图：
  2. 为了清净，直接把HKEY_LOCAL_MACHINE\SOFTWARE\JavaSoft\这个注册表分支给删除了，然后到cmd下，运行java -version，又出现另一个错误：
  3. 解决在Path系统环境变量中，把%JAVA_HOME%\bin调整到最前面

第一步:创建一个表

(
ID NUMBER not null,
AGE NUMBER,

)

第二步:创建序列
