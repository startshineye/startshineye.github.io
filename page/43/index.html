泛型基本概念

  1. 泛型的本质是参数化类型，也就是说所操作的数据类型被指定为一个参数。
  2. 泛型是jdk1.5的所有新特性中最难深入掌握的部分,不过,在实际应用中不能掌握的那么深入，只需要掌握一些基本内容就可以。
  3. 没有使用泛型时，只要是对象,不管是什么类型的对象，都可以存储进同一个集合当中。使用泛型集合，可以将一个集合中的元素限定为一个特定类型。集合中只能存储同一个类型的对象,这样更安全，并且当从集合获取一个对象时,编译器也可以知道这个对象的类型,不需要对对象进行强制类型转换。
  4. Java 泛型（generics）是 JDK 5 中引入的一个新特性, 泛型提供了编译时类型安全检测机制，该机制允许程序员在编译时检测到非法的类型。

BeanUtils操作javaBean

前言

  1. 对javaBean的操作一般是apache提供了专门的工具类对其进行操作那就是beanUtils工具类，
  2. BeanUtils具有强大的功能，能给bean的属性及对象复制，并且能自动进行类型转换。

内容

1.概述
  1. BeanUtils工具由Apache软件基金组织编写，提供给我们使用，主要解决的问题是：把对象的属性数据封装到对象中。在整个J2EE的编程过程中，我们经常会从各种配置文件中读取相应的数据，需要明白的一点是从配置文件中读取到的数据都是String，但是很显然我们的应用程序中不仅仅有String一种数据类型，比如：基本数据类型（int、double、char、float等），还有自定义数据类型（引用数据类型）
  2. 使用其提供的BeanUtils工具，具体的说只需要知道其中的两个方法就能实现类型的转换，很简单，降低了编程的难度。
    2.导包
  3. 想使用别人开发的工具必须将其jar包导入到程序中，但是BeanUtils包存对另外一个软件包的依赖，下面将具体的显示jar包的名字
    commons-beanutils.jar
    commons-logging.jar
    在开始编写具体的代码之前，请将source导入到程序中，否则将不能生成正确的重载函数，只需要使用吧Attach Source，将jar包（使用外部文件的方式导入，不要使用外部文件夹的方式导入）导入即可
    3.使用
    Beanutils工具在使用时几乎只用到以下几个方法。
  4. BeanUtils.setProperty(bean, name, value);其中bean是指你将要设置的对象，name指的是将要设置的属性（写成”属性名”）,value（从配置文件中读取到到的字符串值）
  5. BeanUtils。copyProperties(bean, name, value)，和上面的方法是完全一样的。使用哪个都可以
  6. ConvertUtils.register(Converter converter , ..)，当需要将String数据转换成引用数据类型（自定义数据类型时），需要使用此方法实现转换。
  7. BeanUtils.populate(bean,Map)，其中Map中的key必须与目标对象中的属性名相同，否则不能实现拷贝。
  8. BeanUtils.copyProperties(newObject,oldObject)，实现对象的拷贝

java反射-九-反射实现框架功能

反射实现框架功能

学了那么多反射，究竟反射有什么作用了?
反射一般用在框架的实现方面
在一个类还没有写好之前就已经有可以实现功能了，比如struts框架，在我们还没有写完源程序时候，struts框架就可以运行了。

1.框架与够工具类的区别
  1. 都是别人写的，但是用法不一样，一个是人家调用你，一个是你调用人家。以程序员为主题，你去调用别人的东西就是工具类（你用工具类调用其他），别人调用你的东西叫做框架
    2.框架与框架要解决的核心问题
  2. 开发商做房子卖给客户，客户自己安装门窗和空调。开发商的空房子就是框架，客户做成一个漂亮的房子（一个产品）就需要使用开发商提供的空房子(框架)
  3. 现实开发过程中做的sturs的一个个项目,每个人做的项目都不一样，但是每个人都做的很快，原因就是开始大家就在一个框架下开始干，效率高。
  4. 框架和工具类区别：工具类被用户类调用,而框架则是调用用户提供的类.
    3.框架要解决的核心问题
  5. 我在写框架(房子)时,你这个用户可能还在上小学,还不会写程序？我写的框架程序怎样调用到你以后写的类(门窗)呢？
  6. 因为在写程序时候无法知道要被调用的类名,所以，在程序中无法直接new某个类的实例对象了，用反射解决。
