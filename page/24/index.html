vue(14)-vue生存周期

前言

1.当我们用vue做一个数据回显(也就是当打开或者刷新页面时候在vue方法里面就马上执行这个函数,而不是使用window.onload),我们需要用到vue的生命周期中的一些方法。
2.注意:在我们创建vue对象时候,如果不加

window.onload=function(){
     new Vue({
         el:'#box',
         data:{
             msg:"hello vue"
         }
     });
 }

会报错:

原因就是box在其后面定义,而new Vue()先执行所以找不到,我们也可以将new Vue()放在box后面

内容

1. vue生命周期

要想在vue中实现类似于window.onload效果,我们需要知道vue生命周期:

1.new Vue()其实就是创建一个实例,实例什么时候创建完毕是有标志的钩子函数之一:created
2.将变成正真的数据(vue中的data中的msg中的well)需要进行编译：beforeCompile,Compiled
3.需要把编译好的数据添加到dom树上(文本节点插入到文档中:类似于window.onload)：ready
4.对象也可以销毁:beforeDestroy和destroyed(要想让beforeDestroy和destroyed执行,我们需要绑定事件执行vue里面自带的方法)
5.以上是基于vue1.0,,vue2.0将以上周期函数改名了

1.1钩子函数

created -> 实例已经创建
boforeCompile ->编译之前
compiled ->编译之后
ready ->插入到文档中
beforeDestroy ->销毁之前
destroyed ->销毁之后

2.代码
<!DOCTYPE html>
<html lang="en">
<head>
<meta charset="UTF-8">
<title>vue生命周期</title>
<script src="vue.js"></script>

</head>
  <body>
  <div id="box">
   {{msg}}
  </div>

  <script>
  var vm = new Vue({
       el:'#box',
       data:{
           msg:"well"
       },
       created:function(){
           alert("对象已经创建!");
       },
       beforeCompile:function(){
           alert("编译之前!");
       },
       compiled:function(){
           alert("编译之后");
       },
       ready:function(){
           alert("插入到文档中");
       },
       beforeDestroy:function () {
           alert("销毁前");
       },
       destroyed:function () {
           alert("销毁后");
       }
   });

   //点击文档之后销毁对象,对象的销毁并不会在浏览器里面自动执行,需要事件触发
   document.onclick=function(){
       vm.$destroy();
   }
</script>
</body>
</html>

redis(9)-列表中

前言

redis的插入和删除操作：LSET,LINSERT,LREM,LTRIM

内容

1.设置指定索引上的列表项

LSET key index value

将列表键key索引index上的列表项设置为value,设置成功时命令返回OK。
如果index参数超过了列表的索引 范围,那么命令返回一个错误。

针对表头和表尾节点进行处理时(index为0或者-1)，命令复杂度为O(1);其他情况下,命令的复杂度为O(N),N为列表长度。

2.在指定位置插入列表项

LINSERT key BEFORE|AFTER pivot value

根据命令调用时传递的是BEFORE选项还是AFTER选项,将值value插入到指定列表项pivot的之前或者之后。当pivot不存在于列表key时,不执行任何操作。

返回-1表示pivot不存在;返回0表示键key不存在;插入成功时候则返回列表当前的 长度。

复杂度为O(N),N为列表长度。

3.从列表中删除指定的值

LREM key count value

根据参数count的值,移除列表中与参数value相等的列表项。

  1. 如果count>0,那么从表头开始向表尾搜索,移除最多count个值为value的列表项。
  2. 如果count<0,那么从表尾开始向表头搜索,移除最多count个值为value的列表项。
  3. 如果count=0,那么移除列表中所有值为value的列表项。

命令返回被移除列表项的数量。

命令复杂度为O(N),N为列表的长度。

4.修建列表

LTRIM key start stop

对一个列表进行修剪(trim),让列表只保留指定索引范围内的列表项,而将不在范围内的其他列表项全部删除。两个索引都可以是正数或者负数。

命令执行成功时返回OK,复杂度O(N),N为被移除列表项的数量。

redis(8)-列表上

前言

以有序的方式储存多个可重复的值

内容

1.列表(list)

一个列表可以包含一个或一个以上数量的项(item),每个项按照它们被推入到列表的位置来排序。

每个列表项所处的位置决定了这个项的索引值(index),索引以0为开始,从列表的左端到右端依次递增,位于列表最左端(表头)的项的索引为0,而位于列表最右端(表尾)的项的索引为N-1,其中N为列表长度。

列表包含的项可以出现重复,他们不必是唯一的。

1.1 推入和弹出操作

了解如何向列表添加项,以及如何从列表里面删除项

从列表左端推入值

LPUSH key value [value …]

将一个或以上数量的值依次推入到列表的左端,命令返回新值被推入之后,列表目前包含的项数量。

复杂度O(N),其中N为被推入值的数量,如果只推入一个值,那么命令的复杂度为O(1)。

从列表左端推入多个值

如果执行LPUSH命令时给定多个值,那么各个值将按照给定时的顺序,从左到右依次地被推入到列表左端。

RPUSH key value [value …]

将一个或以上数量的值依次推入到列表的右端,命令返回新值被推入之后,列表目前包含的项数量。

复杂度O(N),其中N依次为被推入值的数量,如果只推入一个值,那么命令的复杂度为O(1)。

获取列表的长度

LLEN key

返回列表key的长度,也即是,返回列表包含的列表项数量。

因为Redis会记录每个列表的长度,所以 这个命令无须遍历列表,它的复杂度为O(1)。

redis(7)-散列下

前言

散列和字符串也有很多相似的地方

内容

1.效果类似命令

2.散列和字符串区别

如果散列键能做的事情,字符串键也能做,那么我们为什么不使用字符串键呢？

2.1散列好处(1):将数据放到同一个地方

1.散列可以将一些相关数据信息存储到同一个地方,而不是直接分散地存储在整个数据库里面,这不仅方便了数据管理,还可以尽量避免误操作发生

2.字符串存储多个值时候会创建多个键,但是对于散列只需要创建一个键,如下:message一个键可以存储散列

2.2散列好处(1):避免键名冲突

在介绍字符串键的时候,我们说过,可以在命名键的时候,使用分隔符来避免命名冲突,但更好的办法是直接使用散列键来储存键值对数据。

举个例子,如果我们需要储存多条消息,那么我们可能会按照格式message::::的方式来创建字符串键并储存数据,比如使用message::12345::receiver键来储存id为12345的消息的接收者,使用message::10086::sender键来储存id为10086的发送者,诸如此类。

但更好的办法是直接使用message::散列键来表示消息信息,并将与消息有关的各项信息储存到散列的各个域里面,比如创建message::12345散列,并将该消息的各项信息分别储存到这个散列的id域,receiver域和sender域里面。

这保证了数据库中每个键的作用都是固定的,单一的,储存的信息都是被隔离的,从而最大限度地键名冲突

2.3散列好处(3):减少内存占用

在一般情况下,保存相同数量的键值对信息,使用散列键比使用字符串键更节约内存。

因为在数据库里面创建的每个键都带有数据库附加的管理信息(比如这个键的类型,最后一次被访问的时间等等),所以数据库里面的键越多,服务器在储存附加管理信息方面耗费的内存就越多,花在管理数据库键上的cpu也会越多

除此之外,当散列保存的域值对数量比较少的时候,Redis会自动使用一种占用内存非常少的数据结构来做散列的底层实现 ,在散列的数量比较多的时候,这一措施对减少内存有很大的帮助。

3.结论

只要有可能的话,就尽量使用散列键而不是字符串键来储存键值对数据,因为散列键管理方便,能够避免键冲突,并且还能节约内存。

一些没办法使用散列键来替代字符串键的情况:
1.使用二进制位操作命令：因为Redis目前支持对字符串键进行SETBIT,GETBIT,BITOP等操作,如果你想使用这些操作,那么只能使用字符串键。
2.使用过期功能:Redis的键过期功能目前只能对键进行过期操作,而不能对散列的域进行过期操作,因此如果你要对键值对数据使用过期功能的话,那么只能把键值对储存在字符串里面。关于过期键的详细信息会在之后介绍。

redis(6)-散列

前言

散列是由多个键值组成

内容

1.散列
  1. 一个散列是由多个域值对(field-value pair)组成,散列的域和值都可以是文字,整数,浮点数或者二进制数据。
  2. 同一个散列里面的每个域必须是独一无二,各不相同的,而域的值没有这一严格要求,换句话说,不同域的值可以是重复的。
  3. 通过命令,用户可以对散列执行设置域值对,获取域的值,检查域是否存在等操作,也可以让Redis返回散列包含的所有域,所有值或者所有域值对。

vue(12)-交互

前言

vue本身框架是不支持交互的,如果vue交互需要通过引入库:vue-resource

内容

  1. 引入vue-resource后，可以基于全局的Vue对象使用http，也可以基于某个Vue实例使用http。

  2. // 基于全局Vue对象使用http
    Vue.http.get(‘/someUrl’, [options]).then(successCallback, errorCallback);
    Vue.http.post(‘/someUrl’, [body], [options]).then(successCallback, errorCallback);

  3. // 在一个Vue实例内使用$http
    this.$http.get(‘/someUrl’, [options]).then(successCallback, errorCallback);
    this.$http.post(‘/someUrl’, [body], [options]).then(successCallback, errorCallback);

  4. 在发送请求后，使用then方法来处理响应结果，then方法有两个参数，第一个参数是响应成功时的回调函数，第二个参数是响应失败时的回调函数。

  5. then方法的回调函数也有两种写法，第一种是传统的函数写法，第二种是更为简洁的ES 6的Lambda写法：

1.传统写法 

this.$http.get('/someUrl', [options]).then(function(response){
// 响应成功回调
}, function(response){
// 响应错误回调
});

2.Lambda写法 

this.$http.get('/someUrl', [options]).then((response) => {
// 响应成功回调
}, (response) => {
// 响应错误回调
});

交互无非几种方式,如下:
1.get
2.post
3.jsonp

1.get
1.1 get请求普通文本

在html同级目录下新建a.txt 

<!DOCTYPE html>
<html lang="en">
<head>
<meta charset="UTF-8">
<title>Title</title>
<script src="vue.js"></script>
<script src="vue-resource.js"></script>
<script>
    window.onload=function(){
        new Vue({
            el:'body',
            data:{

            },
            methods:{
                get:function(){
                    // 在一个Vue实例内使用$http
                    this.$http.get(
                        "a.txt"
                    ).then(function(response){
                        //输出状态码
                        var status = response.status;
                        alert(status);
                        //内容
                        var data = response.data;
                        alert(data);
                    },function(response){
                        alert(response);
                    });
                }
            }
        });
    }
</script>
</head>
<body>
<!--点击按钮获取a.txt里面内容-->
<input type="button" value="get" @click="get()">
</body>
</html>
