package com.yxm.dao;

import java.util.List;

import org.apache.ibatis.annotations.Insert;
import org.apache.ibatis.annotations.Options;
import org.apache.ibatis.annotations.Select;

import com.yxm.entity.Demo;

public interface DemoMapper {

@Select("select * from demo where name = #{name}")
public Demo getByName(String name);

@Select("select * from demo where id = #{id}")
public Demo getById(String id);

@Select("select name from demo where id = #{id}")
public String getNameById(String id);

@Select("select * from demo")
public List<Demo> get();

/**
 * 保存数据.
 */
@Insert("insert into Demo(name) values(#{name})")
@Options(useGeneratedKeys=true,keyProperty="id",keyColumn="id")
public void save(Demo demo);
}

Spring Boot(九)-Spring Boot JdbcTemplate

前言

1.在pom.xml加入jdbcTemplate的依赖
2.编写DemoDao类，声明为：@Repository，引入JdbcTemplate

内容

1.在pom.xml加入jdbcTemplate的依赖
 <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-jdbc</artifactId>
</dependency>

如果在JPA已经加入的话，则可以不用引入以上的配置。
<dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-data-jpa</artifactId>
</dependency>
2.编写DemoDao类，声明为：@Repository，引入JdbcTemplate

如果需要使用JdbcTemplate只需要在需要使用的类中加入：
@Resource
private JdbcTemplate jdbcTemplate;

import com.kfit.demo.bean.Cat;
/**
 * 使用@Repository注解，标注这是一个持久化操作对象.
 * @author yxm
 *
 */
@Repository
public class CatDao {
@Resource
private JdbcTemplate jdbcTemplate;

/**
 * 1、定义一个Sql语句；
 * 2、定义一个RowMapper.
 * 3、执行查询方法.
 */
public Cat select(String catName){
    String sql = "select * from cat where cat_name = ?";
    RowMapper<Cat> rowMapper = new BeanPropertyRowMapper<Cat>(Cat.class);
    return  jdbcTemplate.queryForObject(sql, rowMapper, catName);
}
}
3.编写DemoService类，引入DemoDao进行使用
@Service
public class CatService {
@Resource
private CatDao catDao;
public Cat findCatByCatName(String catName){
    return catDao.select(catName);

Spring Boot(八)-Spring Boot Spring Data JPA介绍

前言

从我们定义的Repository继承CrudRepository,并且@NoRepositoryBean
public interface CrudRepository extends Repository知道,我们需要了解Repository接口即可， Spring Data 的一个核心接口Repository为我们提供了常用的接口，在这节我们就来简单介绍下。

内容

1.Repository接口
1.1特征

Repository 接口是 Spring Data 的一个核心接口，它不提供任何方法，开发者需要在自己定义的接口中声明需要的方法 ：
public interface Repository { }
有这么几点需要强调下：

  1. Repository是一个空接口，即是一个标记接口；
  2. 若我们定义的接口继承了Repository，则该接口会被IOC容器识别为一个Repository Bean纳入到IOC容器中，进而可以在该接口中定义满足一定规范的方法。
  3. 实际上也可以通过@RepositoryDefinition,注解来替代继承Repository接口。
  4. 查询方法以find | read | get开头；
  5. 涉及查询条件时，条件的属性用条件关键字连接，要注意的是条件属性以首字母大写。如：findByNameAndId(String name,String id),name,id就是属性,在查询方法名中大写
    6.使用@Query注解可以自定义JPQL语句实现更灵活的查询。

2.CrudRepository接口
2.1基本特性

CrudRepository 接口提供了最基本的对实体类的添删改查操作

  1. T save(T entity);//保存单个实体
  2. Iterable save(Iterable<? extends T> entities);//保存集合
  3. T findOne(ID id);//根据id查找实体
  4. boolean exists(ID id);//根据id判断实体是否存在
