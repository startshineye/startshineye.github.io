linux(17)-管道符

1.前言
  1. Linux所提供的管道符“|”将两个命令隔开，管道符左边命令的输出就会作为管道符右边命令的输入。连续使用管道意味着第一个命令的输出会作为第二个命令的输入，第二个命令的输出又会作为第三个命令的输入，依此类推。
  2. 命令1 | 命令2 –>命令1的正确输出作为命令2的操作对象
2.命令
  1. netstat命令可以查看所有网络连接
  2. grep查询命令
  3. ps查看进程命令 -aux选项可以查看整个系统的进程

利用管道符查看Tomcat是否启动：
ps -ef|grep ‘tomcat’

ps 查看当前终端的进程
ps -aux 查看整个系统的进程
ps -aux | grep java 通过管道符查看java进程

linux(16)-多命令执行符

多命令执行符

分号(:)–>多个命令顺序执行

逻辑与(&&) –>当命令1执行正确,命令2才会执行,当命令1执行错误,命令2不会执行

逻辑或 -> ||
当命令1执行不正确,才会执行命令2,当命令1执行完毕,命令2不会执行
