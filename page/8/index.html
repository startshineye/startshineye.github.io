传统IO模型:
1.餐厅:我们把整个系统当做一个餐厅
2.大门：serverSocket当做一个大门,监听客户的到达
3.客人:socket客户端
4.服务员:线程,一个服务员就是一个线程
这种情况下服务一个客人就需要一个服务生,餐厅性能和消耗特别大
例图：

NIO模型：
1.餐厅:我们把整个系统当做一个餐厅
2.大门：serverSocketChannel当做一个大门,监听客户的到达
3.客人:socketChannel客户端
4.服务员(最重要):此服务员不是单一服务的,而是一个selector,不仅可以服务客人，还可以监听大门，欢迎新来的客人，相当于一个“黑执事”
如图:

NIO代码分析：
1.先开启一个大门
2.大门交给服务厅监听（将selector注册给serverSocketChannel,会绑定一个selectKey地址）
3.每个客人拿到一个selectKey,处理的时候我们只需要处理key
4.通过serverchannel.accept()接受一个客人,再告诉selector盯紧客人（绑定一个read事件,判断其是否已经点菜）
5.处理点菜的事件read，打印其消息
6.selector.select()监听的事件是isAcceptable(),isReadable())用于区分是来了一个客人或者客人要点菜

Android(39)-Android下主线程

前言

当我们在Android(38)源码查看器下输入http://www.google.com会报以下错误

此时后台会打印出ANR异常:

  1. 1。ANR: Application Not Resource 应用无响应
  2. 2.原因:造成上面的原因是因为:由于在主线程中进行了耗时操作
  3. 3.耗时操作包括:比如连接网络,拷贝大数据,所以主线程不能阻塞。
  4. 4.避免anr 可以把耗时的操作 放到子线程中 自己在创建一个线程
  5. 5，在4.0之后谷歌强制要求连接网络不能在主线程进行访问
    规则:不管你是什么版本的手机,只要做耗时的操作(比如:连接网络 拷贝大的数据 等等)就自己开一个子线程。获取数据后想要更新ui,就使用Handler就可以了

    内容

    1.在主线程中耗时5秒以上实例
  6. 1.ui
    略(参照Android(38))
  7. 2.逻辑
    点击按钮是一个联网操作,我们应该开启资格子线程

  8. 3.测试
    重新部署项目:输入http://www.baidu.com
    报以下异常:


    以上总结:只有主线程(UI线程才可以更新ui)
    但是我们怎样去主线程更改ui呢?也就是怎样把数据从子线程传递到主线程中去,引出handler消息机制。
    2.消息机制(handler)
    1.消息机制使用
  9. 1.使用步骤
    【1】在主线程中定义一个Handler
    private Handler handler = new Handler();
    【2】使用handler会重写handler的里面的handlemessage方法
    public void handemessage(Message msg){};
    【3】拿着我们在主线程创建的handler去子线程发消息
    handler.sendMessage(msg);
    【4】handlermessage方法就会执行 在这个方法里面去更新ui
  10. 2.使用
    在主线程中定义一个handler,并写一个大括号(表示重写此类里面的方法)

    在子线程中发消息,我们先定义一个message

    当一执行完sendMessage则会在主线程中的handleMessage中获取消息,更新ui
    2.消息机制原理

    当我们想要更改ui时候,我们的耗时操作是在子线程的,但是子线程手机不能更改ui的,所以交给了主线程
  11. 1.在子线程中发送消息（比如更改ui），消息是存放在messagequeue
  12. 2.Looper的作用是从messagequeue中取消息,
  13. 3.handler作用是发消息,处理消息
  14. 4.只要不在代码里面创建子线程,任何地方都属于主线程
3.替代handler

在主线程中更改ui时候,我们用到了handler.发送消息给主线程,在主线程中获取消息,然后更改Ui,但是我们也有另一种方法就是不用再主线程更改ui，直接调用runOnUiThread()来替代handler来更改ui.
