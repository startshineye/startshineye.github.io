netty实践(三)-netty之helloword服务端

前言

构建netty服务器分为以下几个步骤:
1.构建一对主从线程组
netty使用Reactor模型,提供了三种线程模型(单线程,多线程,主从多线程),netty官方推荐建议我们去使用主从线程模型
2.定义服务器启动类
任何启动类,不管是tomcat,jetty还是netty都会有一个Bootstrap启动类,这样的server类,我们在netty服务器构建的时候也会创建
3.为服务器设置Channel
之前我们在讲解nio时候,我们说过,当我们的客户端和我们的服务端建立连接之后,就会形成一个双向通道,这个双向通道就是Channel,所以我们在我们的服务器需要定义一个这样的Channel类
4.设置处理从线程池的助手类初始化器
我们会有一个Channel,这个Channel会有一堆的初始化助手类Handler对他进行处理,比如说:编解码处理,读数据写数据等等这样的操作,理解的话可以把它看做一个拦截器Interceptor,我们可以配置多个拦截器去拦截我们的Channel
5.监听启动和关闭服务器
针对于Server,我们需要监听它的启动,监听的话,我们需要设置他的端口,服务启动完了之后,我们也需要做做关闭的监听

2.设置channel初始化器
  1. 每一个channel由多个handler共同组成管道(pipeline)

    1.每一个client和server连接之后都会形成一个channel
    2.每一个channel会有一个pipeline,我们简称是一个管道,那么这个管道会有很多个handler共同组成
    3.上图左侧是一个channel,右侧是一个pipeline,这个pipeline是需要开发者去编写(其实就是一个初始化器),在这个初始化器pipeline中,我们需要为他设置handller，handller就是助手类,助手类就是对channel做一些相应处理,比如先注册HandlerA，然后注册HandlerB
    4.我们可以吧pipeline当成一个大的拦截器,而其他handler就是小的拦截器,请求过来时候我们会一层一层的逐层进行拦截

6.netty扩展
6.1.netty版本

netty版本大致分为:netty3.x 和netty4.x,netty5.x

6.2.netty运用在那些领域

  1. 分布式进程通信
    例如：hadoop,dubbo,akka等具有分布式功能的框架,底层RPC通信都是基于netty实现的，这些框架使用的基本是netty3.x

  2. 游戏服务器
    就java的最新的游戏服务器有部分公司可能已经开始采用netty4.x 或 netty5.x

4.总结
1.SimpleChannelHandler:处理消息的Handler,其中包含连接建立,接发消息,异常处理。

  1. pipeline.addLast(“decoder”,new StringDecoder());对输入字符进行编码,后台只需要String message = (String)e.getMessage();接收。
  2. pipeline.addLast(“encoder”, new StringEncoder());对从服务器达到客户端数据直接输出:ctx.getChannel().write(“hello too”);
  3. StringDecoder()方法继承自ChannelUpstreamHander上行数据输入， new StringEncoder()方法继承自ChannelDownstreamHandler下行数据回显数据。
