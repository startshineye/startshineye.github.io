netty实践(六)-netty5

1.netty5服务端

1.思路

  1. netty5和netty3思路上是有类似的
    1.创建服务–>创建引导服务
    2.创建线程池–>创建事件循环组
    3.设置socket工厂–>设置channel类型
    4.设置管道工厂–>设置会话处理器
    5.监听端口
    6.连接关闭

  2. 说明
    1.netty3中是直接使用线程池,而netty5是更大的封装,里面是封装了线程池,使用了EventLoopGroup,一个EventGroup就代表一个线程
    2.设置管道工厂是直接设置子处理器
    3.boss是用于监听端口的
    4.设置socket工厂：服务端:NioServerChannel 客户端：NioChannel
    4.设置管道:服务端:childHandler 客户端Handler

2.netty5客户端

1.思路
  1. 思路
    1.创建客户端服务
    2.创建线程池
    3.创建socket工厂
    4.创建pipeiline工厂
    5.建立连接,返回channel
    6.通过channel返回数据
  2. 注意
    1.netty5中BootStrap代表客户端引导服务,ServerBootStrap代表服务端引导，区别于netty3.
    2.boss是用来监听端口的线程

    import io.netty.channel.ChannelHandlerContext;
import io.netty.channel.SimpleChannelInboundHandler;
/**
 * netty5客户端
 * @author yxm
 *
 */
public class ClientHandler extends SimpleChannelInboundHandler<String>{
@Override
protected void messageReceived(ChannelHandlerContext ctx, String msg)
        throws Exception {
 System.out.println("服务端消息："+msg);
 ctx.channel().writeAndFlush("客户端: hi");
}
/**
 * 新客户端接入
 */
@Override
public void channelActive(ChannelHandlerContext ctx) throws Exception {
    System.out.println("channelActive");
}

/**
 * 客户端断开
 */
@Override
public void channelInactive(ChannelHandlerContext ctx) throws Exception {
    System.out.println("channelInactive");
}

/**
 * 异常
 */
@Override
public void exceptionCaught(ChannelHandlerContext ctx, Throwable cause) throws Exception {
    cause.printStackTrace();
}
}
