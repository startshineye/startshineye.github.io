redis(6)-散列

前言

散列是由多个键值组成

内容

1.散列
  1. 一个散列是由多个域值对(field-value pair)组成,散列的域和值都可以是文字,整数,浮点数或者二进制数据。
  2. 同一个散列里面的每个域必须是独一无二,各不相同的,而域的值没有这一严格要求,换句话说,不同域的值可以是重复的。
  3. 通过命令,用户可以对散列执行设置域值对,获取域的值,检查域是否存在等操作,也可以让Redis返回散列包含的所有域,所有值或者所有域值对。

2.基本操作

基本操作:关联域值对,获取域的值,检查域是否存在,获取域值对数据是否存在,等等。

2.1关联域值对

HSET key field value

在散列键key中关联给定的域值对field和value。如果域fiield之前没有关联值,那么命令返回1；如果域field已经有关联值,那么命令用新值覆盖旧值,并返回0。

2.2 获取域所关联的值

HGET key filed

返回散列键key中,域filed所关联的值。如果域filed没有关联值,那么返回nil

复杂度O(1)

2.3仅当域不存在时，关联域值对

HSETNX key filed value

如果散列键key中,域filed不存在(也即是,还没有与之关联的值),那么关联给定的域值对filed和value。

如果域field已经有与之相关联的值,那么命令不做动作,返回为0。

复杂度O(1)

2.4检查域是否存在

HEXISTS key filed

查看散列键key中,给定域是否存在;存在返回1，不存在返回0。

复杂度O(1)

2.5 删除给定的域值对

HDEL key filed [feild …]

删除散列键key中的一个或者多个指定域,以及那些域的值。不存在的域将会被忽略。命令返回被成功删除的域值对数量。

复杂度为O(N),N为被删除的域值对数量。

2.6 获取散列包含的键值对数量

HLEN key

返回散列键key包含的域值对数量

复杂度O(1)

为什么命令叫hkeys而不是hfields?
对于散列来说,key和field表示的是同一个意思,并且key比field更容易拼写。所以redis选择使用keys来做命令的名字,而不是hfields。
