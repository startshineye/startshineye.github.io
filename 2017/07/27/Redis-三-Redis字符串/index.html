Redis(三)-Redis字符串

字符串

1.理论

1.字符串是Redis中最简单的数据结构,它既可以存储文字(比如:”hello word”),又可以储存数字(比如整数10086和浮点数3.14),还可以存储二进制数据(比如10010100)。
2.Redis为这几种类型的值分别设置了相应的操作命令,让用户可以针对不同的值做不同的处理。

2.基本操作
2.1 SET key value

首先启动redis服务:

打开客户端:

1.命令：SET key value
将字符串键key的值设置为value,命令返回OK表示设置成功。
如果字符串键key已经存在,那么用新值覆盖原来的旧值。
复杂度O(1)。
键key相同时候将会覆盖掉原来的value

2.2 SET key value[NX|XX]

SET 命令还支持可选的NX选项和XX选项:

  1. 如果给定NX选项,那么命令仅在键key不存在的情况下,才进行设置操作,如果键key已经存在,那么SET…NX命令不做动作(不会覆盖旧值)。
  2. 如果给定XX选项,那么命令仅在键key存在的情况下,才进行设置操作,如果键key不存在,那么SET…XX命令不做动作(一定会覆盖旧值)。

在给定NX选项和XX选项的情况下,SET命令在设置成功时候返回OK,设置失败时返回nil(返回为null,类似于null)。

2.3 获取字符串的值:GET key

GET key
返回字符串key储存的值
复杂度O(1)

2.4 示例:使用Redis来进行缓存

我们可以使用Redis来缓存一些经常会被用到,或者需要消耗大量资源的内容,通过将这些内容放到Redis里面(也即是内存里面),程序可以以极快的速度取得这些内容。

举个例子,对于一个网站来说,如果某个页面经常会被访问到,或者创建页面时候消耗的资源比较多(比如需要多次访问数据库,生成时间比较长,等等),那么我们可以使用Redis将这个页面缓存起来,减轻网站的负担,降低网站的延迟值。

2.6 仅在键不存在的情况下进行设置 SETNX key value

SETNX key value

仅在键key不存在的情况下,将键key的值设置为value,效果和SET key value NX一样。NX的意思为”Not eXists”(不存在)

键不存在并且设置成功时,命令返回1；因为键已经存在而导致设置失败时,命令返回0。

复杂度O(1)。

3.0 一次设置多个不存在的键

3.1 设置新值返回旧值

GETSET key new-value

将字符串键的值设置为new-value,并返回字符串键在设置新值之前存储的旧值(old-value)

复杂度为O(1)

3.2 追加内容到字符串末尾

APPEND key value

将值value推入到字符串键key已存储内容的末尾。

O(N),其中N为被推入值的长度。

3.3 返回值的长度

STRLEN key

返回字符串键key存储的值的长度

因为redis会记录每个字符串值的长度,所以获取该值的复杂度为O(1)
