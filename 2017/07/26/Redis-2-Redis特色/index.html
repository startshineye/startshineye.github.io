Redis(2)-Redis特色

前言

内容

1.独特的键值对模型

1.很多数据库只能处理一种数据结构:

  1. SQL 数据库–表格
  2. Memcached–键值对数据库,键和值都是字符串
  3. 文档数据库(CouchDB,MongoDB)–由JSON/BSON组成的文档

2.而一旦数据库提供的数据结构不适合去做某件事情时候,程序写起来就会非常麻烦。比如一般的oracle和mysql的并行操作时候需要两张表的关联(join-on)

3.Redis也是键值对数据库,但是和Memcached不同的是,Redis的值不仅可以是字符串,它还可以是其他五种数据结构中的任意一种。

4.通过选用不同的数据结构,用户可以使用Redis解决各式各样的问题

2.内存存储,速度极快

Redis将数据存储在内存里面,读写数据的时候都不会受到硬盘I/O速度的限制,所以速度极快

硬盘数据库工作模式:SQL数据库,工作方式是将值存储到硬盘中,将索引存储到内存中。当数据库去查找某个值的时候,先去内存中找到值所对应的索引,然后根据索引去找到存储到硬盘中的数据,问题:在读取写入硬盘的时候,由于速度比较慢,读写比较多的时候,将会导致阻塞或者等待比较长的时间

内存数据库工作模式:直接把值存放在内存中,内存数据库直接通过内存访问 值

3.丰富的附加功能

1.持久化功能:将存储在内存里面的数据保存到硬盘里面,保障数据安全,方便进行数据备份和恢复（redis服务一关电或者断点内存中数据就会没有，所以就需要通过持久化将数据进行备份）
2.发布与订阅功能:将消息同时分发给多个客户端,用于构建广播系统(聊天室中运用挺多)
3.过期键功能:为数据库键设置一个过期时间,让它在指定时间之后自动删除
4.事物功能:原子地执行多个操作,并提供乐观锁功能,保证处理数据时候的安全性.
5.脚本功能:在服务器端原子地执行多个操作,完成复杂的 功能,并减少客户端与服务器端通信往返次数(多次客户端的操作写在脚本里面,然后一次性在服务器端执行)
6.复制:为指定的redis服务器创建一个或多个复制品,用于提升数据安全性,并分担读请求的负载
7.Sentinel：监控Redis服务器状态,并在服务器发生故障时候,进行自动故障转移.
8.集群:创建分布式数据库,每个服务器分别执行一部分写操作和读操作

4.完善的文档

Redis具有完善,易读的文档,加上Redis本身功能的简单性，就算是新手也可以轻松上手
以下是官网上为每一个命令都做了详细文档.

5.良好的支持

1.antire非常勤奋,在每个版本都会不断增加有用的新功能:

  1. 2.6新增脚本功能,并为很多命令添加了多参数支持(比如:SADD,ZADD,等等);
  2. 2.8添加了数据库通知功能,HyperLogLog数据结构以及SCAN命令,实现了部分重同步;
  3. 3.0将会推出稳定版的Redis集群,另外还有很多新功能在持续开发中…

2.Bug一旦出现就会被修复,齐全的测试套件以及稳扎稳打的开发策略,使得Redis健壮可靠

3.有问题时候,在Redis论坛发帖,或者到Redis的GitHub页面发issue,又或者和作者联系

4.Pivotal公司雇佣antirez全力开发Redis,无后顾之忧，并提供专门的Redis开发和维护咨询工作

5.阿里云，百度云，Amazon,RedisLab等公司都提供了基于Redis的应用服务

6.广泛使用

1.Twitter使用Redis来存储用户时间线
2.StackOverflow使用Redis来进行缓存和消息分发
3.Pinterest使用Redis来构建关注模型(follow model)和兴趣图谱(iinterest graph)
4.Flickr使用redis构建队列
5.GitHub使用Redis作为持久化的键值对数据库,并使用Resque来实现消息队列
6.新浪微博使用Redis来实现计数器,反方索引,排行榜，消息队列，存储用户关系
7.知乎使用Redis来进行计数，缓存，消息分发和任务调度
