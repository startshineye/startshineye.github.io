java并发编程(二)-脏读

脏读

对于对象的同步和异步的方法,我们在设计的时候,一定要考虑问题的整体,不然就会出现数据不一致性的错误,很经典的错误就是脏读

2.代码

结果:
getValue最终结果:name= zs age=23
setValue最终结果:name= ls age=24

结果:
setValue最终结果:name= ls age=24
getValue最终结果:name= ls age=24

3.总结
  1. 在setValue的时候不希望其它线程去getValue
  2. 分析:建立一个类,添加set(同步),get方法,set方法中设值后延时2秒,在主线程中创建一个对象并调用set，延时一秒在get
  3. 代码有两个线程:主线程和t1线程,主线程一直往下面执行,t1线程执行后延时2秒
  4. 以上t1在设值的时候有主线程打扰,以上是一个对象,一把锁,t1先拿了那把锁,操作对象,此时主线程也要访问加了锁的柜子,所以要等到t1释放那把锁
  5. 在我们对一个对象的方法加锁的时候,需要考虑业务的整体性,即为setvalue/getvalue方法同时加上synchronzied同步关键字,保证业务的原子性,不然就会出现业务错误

Oracle数据库

场景:A在9:00时候我们全盘扫描查询数据库里面值,我们在9:10时候才查询相互值100,B在9:05时候进行DML操作修改值为:200,问在9:10时候查询出的值是多少?
答案：是100,这是由于Oracle的”一致性读”有一个”undo”原理,在进行修改时候,会把100传入undo里面,然后9:10取出来的值也是100
