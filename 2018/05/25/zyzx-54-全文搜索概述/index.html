zyzx(54)-全文搜索概述

前言

场景:
1.一般我们在浏览器中使用搜索引擎搜索问题
2.我们查询关系型数据库时候,本身也是搜索过程,根据sql语句,数据库把搜索到的结果返回给查询结果集

主要内容:
1.全文搜索
2.全文搜索产品(ElasticSearch简介)
3.ElasticSearch核心概念
4.ElasticSearch与SpringBoot集成
5.ElasticSearch实战

内容

1.全文搜索
1.1数据结构
  1. 结构化：指具有固定格式或有限长度的数据,如数据库,元数据等,(见得最多的就是每次我们创建数据库时候需要制定数据的格式和长度)
  2. 非结构化:指不定长或者无固定格式的数据,如:邮件,word文档等。

结构化的数据查询一般来说我们都是比较熟悉的,但是对于非结构化的数据,我们如何查询呢？

1.2 非结构化数据的检索

1.顺序扫描法(Serial Scanning)
比如在文档里面查找字符串,就是从头到尾找此字符串,直到找到为止(类似从字典里面查找某一个字的时候,我们从头到尾的找),这种方法对于数据量小的数据量,效率很高,但是数据量很大,就会很慢,那么大文件数据用哪种搜索法比较好呢?

2.全文搜索(Full-text Search)
对于数据量比较大的时候,此方法合适
全文搜索:就是把非结构化的部分信息提取出来重新进行组织,使其变得有结构,对有结构的数据进行搜索,从而达到搜索相对较快的目的

非结构化数据的检索其实就是把非结构化的数据组织成结构化数据,然后创建索引,来实现搜索目的.

索引:从非结构化数据中部分提取出的结构化数据

比如字典:里面有很多汉字,那么如何找到其中的一个汉字,全文搜索就是替每一个汉字都建立起一套规则,汉字的拼音 就是索引

1.3 全文搜索概念

全文搜索概念:全文搜索是一种将文件中所有文本与搜索 项匹配的文字资料检索方法

1.4 全文搜索实现原理

以从字典中搜索出汉字为例:
建立文本库:建立要搜索的数据源
建立索引:提取结构化数据
执行搜索:进行搜索,结果返回给用户
过滤结果:将返回结果给用户时候,有很多种结果,在一个页面里面不可能全部展现出现,应该分页展示

1.5 全文搜索实现技术

基于Java的开源实现
1.Lucene:基于全文搜索的引擎
2.ElasticSearch:基于Lucene之上的全文搜索
3.Solr:与ElasticSearch同类型的产品,也是基于Lucene的

以车为例:Lucene其实就是车的发动机,对于具体什么车,ElasticSearch和Solr就是具体的车:比如奥迪,大众,可能用到了同一个引擎(Lucene)

ElasticSearch与Solr区别:
1.Solr在分布式管理上:利用了zookeper进行分布式,并且支持很多种数据格式:比如xml，json,csv,在传统业务上要好于Solr，但是在实时性处理要低ElasticSearch
2.ElasticSearch自带分布式,格式只支持json来提供restful接口
3.在目前市场上:ElasticSearch主流些,用的比较多

2.全文搜索产品(ElasticSearch简介)

ElasticSearch是什么?

  1. 高度可扩展的开源全文搜索和分析引擎(不单单是搜索,还提供很多分析功能)
  2. 快速地,近实时地对大数据进行存储,搜索和分析
  3. 用来支撑有复杂的数据搜索需求的企业级应用
    2.1 ElasticSearch特点
  4. 分布式:分布式搜索引擎,每个索引实现分片,每个分片,有很多副本数据分不到多个分片上
  5. 高可用:正因为是分布式造就了高可用,单应用崩溃爱概率比较大
  6. 多类型:支持多种数据类型
  7. 多API:支持java原始api,也支持restful结构类型的API
  8. 面向文档:不需要事先定义一些模式
  9. 异步写入:性能比同步更高
  10. 近实时:实时性好
  11. 基于lucene
  12. Apache协议
3.ElasticSearch核心概念
  1. 近实时:近实时是接近实时,但是不是实时，从搜索文档,到可搜索有一个轻微延时,大概是1秒,为什么会有这轻微延时呢？Lucene其实是可以做到这真实的实时的,但是如果做到实时,要么是牺牲索引,每次搜索 都得刷新数据,要么是牺牲查询的效率,每次查询都得做一个刷新,所以做了一个折中,每个n秒做一个刷新
  2. 集群:一个或者多个节点的集合,保存应用全部数据,并提供全部数据的索引和集成的数据,每个集群都要有一个名称,默认是ElasticSearch名称
  3. 节点:集群中的单台服务器,保存数据,参与整个集群的索引和搜索操作
  4. 索引:加快搜索,文档得集合
  5. 类型：索引中包含文档得细分
  6. 文档:进行索引的基本单位,与索引中的类型是对应的，文档对应于结构化数据的实体,文档得实例对应于实体的实例,也就是具体的数据。
  7. 分片:ES把索引分片,然后用分片来存储部分数据
  8. 副本:分片可以设置各自副本,设置副本目的：故障不可避免,为了是系统高可用,我们需要副本,放到多节点上

####### 1.文档EsBlog
ES中id不是我们普通所指的Long类型的id,而是String类型id 

@Document(indexName = "blog",type = "blog")//标识是一个文档:indexName 索引名称
public class EsBlog implements Serializable{
private static final long serialVersionUID = 1L;
/**
 * Es主键是String类型的
 */
@Id //标识是主键
private String id;
/**
 * 标题
 */
private String title;
/**
 * 摘要
 */
private  String summary;
/**
 * 内容
 */
private  String content;

protected EsBlog(){//JPA规范要求,防止直接使用

}
public EsBlog(String title,String summary,String content){
  this.title = title;
  this.summary = summary;
  this.content = content;
}

public String getId() {
    return id;
}

public void setId(String id) {
    this.id = id;
}

public String getTitle() {
    return title;
}

public void setTitle(String title) {
    this.title = title;
}

public String getSummary() {
    return summary;
}

public void setSummary(String summary) {
    this.summary = summary;
}

public String getContent() {
    return content;
}

public void setContent(String content) {
    this.content = content;
}

@Override
public String toString() {
    return "EsBlog{" +
            "id='" + id + '\'' +
            ", title='" + title + '\'' +
            ", summary='" + summary + '\'' +
            ", content='" + content + '\'' +
            '}';
   }
}

####### 2.资源库 EsBlogRepository
定义一个方法:只从title,summary,content里面取搜索,只需要有一个地方搜索到,就返回此结果 

public interface EsBlogRepository extends ElasticsearchRepository<EsBlog,String>{
Page<EsBlog> findDistinctEsBlogByTitleContainingOrSummaryContainingOrContentContaining(String title,String summary,String content,Pageable pageable);
}

####### 4.BlogController(控制器) 

@RestController
@RequestMapping("/blogs")
public class EsBlogController {
@Autowired
private EsBlogRepository esBlogRepository;
@GetMapping
public List<EsBlog> list(@RequestParam(value="title",required=false,defaultValue="") String title,
                         @RequestParam(value="summary",required=false,defaultValue="") String summary,
                         @RequestParam(value="content",required=false,defaultValue="") String content,
                         @RequestParam(value="pageIndex",required=false,defaultValue="0") int pageIndex,
                         @RequestParam(value="pageSize",required=false,defaultValue="10") int pageSize) {
    // 数据在 Test 里面先初始化了，这里只管取数据
    Pageable pageable = new PageRequest(pageIndex, pageSize);
    Page<EsBlog> page = esBlogRepository.findDistinctEsBlogByTitleContainingOrSummaryContainingOrContentContaining(title,summary,content,pageable);
    return page.getContent();
   }
}
