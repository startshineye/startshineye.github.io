java放射(六)-反射调用main方法

用反射方式执行某个类中的main方法

1.目标

写一个程序，这个程序能够根据用户提供的类名，去执行该类中的main方法。用普通方式调用完后，为什么要用反射方式去调用呢？

2.问题

启动java程序的main方法的参数是一个字符串数组,即是public static void main(String[] args),通过反射方式来调用这个mian方法时，如何为invoke方法传递参数呢？jdk1.5的语法，整个数组是一个参数，jdk1.4语法，数组中的每一个元素对应一个参数,当把一个字符串数组作为参数传递给invoke方法时候，javac底层会按照哪种语法进行处理呢？jdk1.5肯定要兼容jdk1.4的语法，会按照jdk1.4的语法进行处理，即把数组打散成为若干个单独的参数。所以，在给main方法传递参数时候，不能使用代码mainMethod.invoke(null,new String[]{“xxx”}),javac只把他当做jdk1.4的语法进行理解,而不把它当做jdk1.5的语法理解，因此会出现类型不匹配的问题。

3.解决办法
  1. mainMethod.invoke(null,new Object[]{new String[]{“xxx”});
  2. mainMethod.invoke(null,(Object)new String[]{“xxx”});编译器会做特殊处理，编译时不把参数看做数组对待，也就不会数组打散成若干个参数。
