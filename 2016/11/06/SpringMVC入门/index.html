一:SpringMVC介绍

1.1. SpringMVC简介

  • SpringMVC也叫Spring web mvc,是一种基于Java的实现了MVC架构模式的思想的请求驱动类型(请求-响应模型)的轻量级Web框架,将web层进行职责解耦.
  • SpringWebMVC是服务到工作者模式的优化实现。前端控制器DispatcherServlet；应用控制器其实拆为处理器映射器(Handler Mapping)进行处理器管理和视图解析器(View Resolver)进行视图管理；页面控制器/动作/处理器为Controller接口（仅包含ModelAndView handleRequest(request, response) 方法）的实现；
  • SpringMVC是在Spring3.0后发布的

    由以上Spring的结构图可以知道,Spring由四部分组成:Dao部分(DAO与ORM),AOP部分,Web部分(JEE与Web),及IoC容器(Core).

1.2 SpringMVC请求处理流程

SpringMVC请求处理流程如下图所示:区别于SpringMVC执行流程

具体执行步骤如下：
  1. 用户发送请求到前端控制器:前端控制器根据请求信息（如URL）来决定选择页面控制器/处理器进行处理如图上的1、2步骤；
  2. 页面控制器接/处理器收到请求后，进行功能处理，首先需要收集和绑定请求参数到一个对象，这个对象在Spring Web MVC中叫命令对象，并进行验证，然后将命令对象委托给业务对象进行处理；处理完毕后返回一个ModelAndView（模型数据和逻辑视图名）；上图中的3、4、5步骤；
  3. 前端控制器根据返回的逻辑视图名选择相应的视图进行渲染，并把模型数据传入以便视图渲染图上的6、7；
  4. 前端控制器再次将响应返回给用户图上的8；至此整个结束。

1.3 SpringMVC程序执行流程

想要学好Spingmvc,第一必须要知道Springmvc执行流程:

核心架构的具体流程步骤如下：
  1. 浏览器/用户发送请求到DispatcherServlet(中央调度器/前端控制器)，
  2. DispatcherServlet(中央调度器/前端控制器)直接把请求转给解析器进行处理(不做任何处理)
  3. 处理器映射器会根据请求,找到处理该请求的处理器,并将其封装为处理器执行链(HandlerExecutionChain)对象（包含一个Handler处理器（页面控制器）对象、多个 HandlerInterceptor拦截器）对象，并返回给中央调度器；
  4. DispatcherServlet——>HandlerAdapter中央调度器根据处理器执行链链中的处理器，找到能执行该处理器的适配器(HandlerAdapter)，从而支持多种类型的处理器.
  5. HandlerAdapter——>处理器功能处理方法的调用，HandlerAdapter将会根据适配的结果调用真正的处理器的功能处理方法，完成功能处理；
  6. 处理器将处理结果封装成ModelAndView对象（包含模型数据、逻辑视图名）[处理器业务是在适配器里面执行的]；
  7. 处理器适配器(HandlerAdapter)直接将结果返回给中央调度器(DispatcherServlet).
  8. 中央调度器(DispatcherServlet)调用视图解析器(ViewResolver),将ModelAndView中的视图名称封装成视图对象.
  9. 视图解析器(ViewResolver)将封装了的视图对象返回给中央调度器(DispatcherServlet).
  10. 中央调度器(DispatcherServlet)调用视图对象,将其进行渲染,即进行数据填充,形成响应对象.
  11. 中央调度器(DispatcherServlet)响应浏览器.
由以上具体流程开发步骤我们可以具体的核心开发配置步骤：
  • DispatcherServlet在web.xml中的部署描述，从而拦截请求到Spring Web MVC
  • HandlerMapping的配置，从而将请求映射到处理器
  • HandlerAdapter的配置，实现处理器和适配器的映射从而支持多种类型的处理器
  • 处理器（页面控制器）的配置，从而进行功能处理
  • ViewResolver的配置，从而将逻辑视图名解析为具体视图技术

2.1 导入Jar包

创建web项目后,首先导入Jar包.SpringMVC需要的基本jar包,是在原Spring的Jar包基础上增加两个Jar包.
Spring-context-support:包含支持UI模板,邮件服务,缓存Cache等方面的类.Spring-webmvc:对SpringMVC的实现.

2.2 中央调度器(DispatcherServlet)的配置

当用户发送请求给中央处理器,服务器解析web.xml根据请求找到中央处理器,所以我们必须配置web.xml如下:

  • 全限定性类名: 该中央调度器为一个Servlet，名称为DispatcherServlet。中央调度器的全限定性类名在导入的Jar 文件spring-webmvc-4.2.1.RELEASE.jar 的第一个包org.springframework.web.servlet下可找到。
  • load-on-startup: 在中添加的作用是，标记是否在Web服务器(这里是Tomcat）启动时会创建这个Servlet 实例，即是否在Web 服务器启动时调用执行该Servlet 的init()方
    法，而不是在真正访问时才创建。它的值必须是一个整数。
    1. 当值大于等于0 时，表示容器在启动时就加载并初始化这个servlet，数值越小，该Servlet的优先级就越高，其被创建的也就越早；
    2. 当值小于0 或者没有指定时，则表示该Servlet 在真正被使用时才会去创建。
    3. 当值相同时，容器会自己选择创建顺序。
  • url-pattern:不能写为/*，最好也不要写为/，建议写为*.action 的形式
  • 配置文件位置与名称:
    注册完毕后，可直接在服务器上发布运行. MyEclipse 控制
    台均会抛出FileNotFoundException 异常。即默认要从项目根下的WEB-INF 目录下找名称为
    Servlet 名称-servlet.xml 的配置文件。这里的“Servlet 名称”指的是注册中央调度器时
    标签中指定的Servlet 的name 值。本例配置文件名为springmvc-servlet.xml

三:总结

  • SpringMVC流程是怎样的?SpringMVC入门主要把握其执行流程,整个SpringMVC基本就是按照流程展开的
  • 理清流程中五大器是怎么布置的?( DispatcherServlet,HandlerMapping, HandlerAdapter,Hander, ViewResolver)
但发现一个问题：在流程介绍中所述的

重要的处理器映射器、处理器适配器、视图解析器都在哪里，不用做配置吗？这些内容均在DispatcherServlet 的默认配置DispatcherServlet.properties
文件中被定义。这个文件与DispatcherServlet 类在一个包下，而且是当Spring 配置文件中
没有指定配置时使用的默认情况。


至此我们可以总结出:五大器部署情况
  • DispatcherServlet就是一个Servlet所以在web.xml文件中配置
  • HandlerMapping默认情况下在DispatcherServlet 的默认配置DispatcherServlet.properties
    文件中被定义。,
  • HandlerAdapter, 默认情况下在DispatcherServlet 的默认配置DispatcherServlet.properties
    文件中被定义。,
  • Hander,在配置文件中定义(默认情况下id与路径关联)
  • ViewResolver 默认情况下在DispatcherServlet 的默认配置DispatcherServlet.properties
    文件中被定义。,
